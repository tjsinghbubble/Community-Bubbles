# Bubble Sharing System

## Purpose

This document describes how to implement **shareable short URLs for Bubbles**.

The goal is to allow users to easily share a bubble with others using a **short, stable link** that works across web, mobile apps, and QR codes.

Example share link:

```
https://app.com/b/a7F3kP
```

Where:

* `b` → bubble route
* `a7F3kP` → unique short identifier for a bubble

---

# System Design Goals

The sharing system should provide:

* Short and clean URLs
* Stable identifiers
* Mobile deep linking compatibility
* QR code support for physical locations
* Simple backend implementation
* Scalability to millions of bubbles

---

# URL Structure

Bubble URLs follow this format:

```
https://app.com/b/{shortId}
```

Example:

```
https://app.com/b/a7F3kP
```

The **shortId uniquely identifies the bubble** and is used for database lookup.

---

# Database Changes

The existing `bubbles` table must include a `short_id` column.

## Migration

```sql
ALTER TABLE bubbles ADD COLUMN short_id TEXT UNIQUE;
```

Create an index for fast lookup:

```sql
CREATE UNIQUE INDEX idx_bubbles_short_id
ON bubbles(short_id);
```

# Short ID System

## Base62 Character Set

Short IDs are generated using Base62 characters.

```
0123456789
abcdefghijklmnopqrstuvwxyz
ABCDEFGHIJKLMNOPQRSTUVWXYZ
```

## ID Length

```
6 characters
```

Total combinations:

```
62^6 = 56+ billion
```

This is sufficient for a very large number of bubbles.

---

# Utility: Short ID Generator

Create the following file:

```
utils/shortId.js
```

Implementation:

```javascript
const chars =
  "0123456789abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ";

export function generateShortId(length = 6) {
  let result = "";

  for (let i = 0; i < length; i++) {
    result += chars.charAt(Math.floor(Math.random() * chars.length));
  }

  return result;
}
```

Example generated IDs:

```
a7F3kP
K91pLx
Z4m2Qa
```

# Bubble Creation Logic

When creating a bubble:

1. Generate a short ID
2. Check if the ID already exists
3. Retry if collision occurs
4. Save the ID in the bubble record

Example implementation:

```javascript
import { generateShortId } from "../utils/shortId.js";

async function createBubble(name, otherFields) {
  let shortId;
  let exists = true;

  while (exists) {
    shortId = generateShortId(6);

    const result = await db.query(
      "SELECT 1 FROM bubbles WHERE short_id = ?",
      [shortId]
    );

    exists = result.length > 0;
  }

  const bubble = await db.query(
    `INSERT INTO bubbles (name, short_id)
     VALUES (?, ?)
     RETURNING *`,
    [name, shortId]
  );

  return bubble;
}
```

export function getBubbleShareUrl(bubble) {

# Bubble Resolution Route

Add a route to resolve the short URL.

Endpoint:

```
GET /b/:shortId
```

Example request:

```
GET /b/a7F3kP
```

Implementation:

```javascript
app.get("/b/:shortId", async (req, res) => {
  const shortId = req.params.shortId;

  const bubble = await db.query(
    "SELECT * FROM bubbles WHERE short_id = ?",
    [shortId]
  );

  if (!bubble) {
    return res.status(404).send("Bubble not found");
  }

  return res.json(bubble);
});
```

In production this route may:

* redirect to a bubble web page
* open a mobile deep link
* return bubble metadata

# Security Considerations

Short IDs should:

* be random
* not expose internal IDs
* not allow enumeration attacks

6-character Base62 IDs provide **56+ billion combinations**, making guessing difficult.

If needed later, rate limiting can be added to the `/b/:shortId` endpoint.
